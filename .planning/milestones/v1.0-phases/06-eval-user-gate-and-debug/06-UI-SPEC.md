# Phase 6 -- UI Design Contract

> Output formatting, interaction, and copywriting contract for the Eval, User Gate, and Debug phase. This phase has NO browser UI -- all user-facing output is structured markdown appended to existing verify reports, MCP JSON responses, CLI progress messages, and interactive gate prompts rendered in Claude Code's terminal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (CLI plugin, no frontend) |
| Icon library | not applicable |
| Font | not applicable (terminal/markdown output) |

**Rationale:** CodeScope is a Claude Code plugin. Phase 6 outputs are: (1) eval and debug cycle sections appended to the existing verify report on disk, (2) interactive user gate prompts in the terminal, (3) MCP JSON responses from codescope_eval, (4) CLI progress lines. There are zero browser-rendered components.

---

## Spacing Scale

Not applicable. This phase produces structured markdown and JSON, not visual layouts.

Markdown report structure uses the same conventions established in Phase 5:
- One blank line between sections
- No blank line between list items within a section
- Two blank lines before H2 headings
- One blank line before H3 headings

---

## Typography

Not applicable. Terminal and markdown output inherit the consumer's rendering.

Markdown heading hierarchy for eval/debug report sections (appended to existing verify report):

| Level | Usage |
|-------|-------|
| H2 (`##`) | `Eval Results`, `Debug Cycle N` -- major sections appended to verify report |
| H3 (`###`) | Individual criteria results: `Scope Compliance`, `Convention Adherence`, `Completeness`, `Correctness` |
| Bold (`**`) | Status labels (`PASS`/`FAIL`/`SKIPPED`), severity tags (`ERROR`/`WARN`/`INFO`), gate actions (`APPROVED`/`IGNORED`/`DEFERRED`) |
| Code (`` ` ``) | File paths, function names, config keys, convention names, criterion names |
| Lists (`- `) | Findings within a criterion, fix plan tasks, design decision options |

---

## Color

Not applicable. No visual rendering layer.

Severity markers reuse the Phase 5 text label system (no color codes):

| Severity | Marker | Meaning | Gate Routing |
|----------|--------|---------|--------------------|
| ERROR | `[ERROR]` | Scope violation, missing implementation, broken correctness | Must debug (all modes) |
| WARN | `[WARN]` | Convention violation, incomplete implementation, design decision | Auto-debug in auto-skip-minor; user choice in interactive |
| INFO | `[INFO]` | Minor style suggestion, optional improvement | Auto-skipped in auto-skip-minor; user choice in interactive |
| SKIPPED | `[SKIPPED]` | Criterion disabled in config | No action needed |
| PASS | `[PASS]` | Criterion passed with zero ERROR findings | No action needed |

Source: CONTEXT.md D-02, D-04, D-09. Extends Phase 5 05-UI-SPEC.md severity table.

## Copywriting Contract

### Primary CTA

| Element | Copy |
|---------|------|
| Eval trigger (automated, within orient pipeline) | `## Evaluating changes...` |
| Eval trigger (standalone MCP) | `codescope_eval` tool call with `files` and optional `checks` parameters |

### Eval Report Section (appended to verify report per D-19)

```
## Eval Results

**Evaluator:** {model_name}
**Criteria evaluated:** {N}/{total}
**Duration:** {duration}

### Scope Compliance

**Verdict:** [PASS] / [FAIL]

- [ERROR] `{file}:{line}` -- {description}
  Evidence: {specific code or verify data}
- [WARN] `{file}:{line}` -- {description}
  Evidence: {evidence}

### Convention Adherence

**Verdict:** [PASS] / [FAIL]

- [WARN] `{file}:{line}` -- Violates `{convention-name}`
  Evidence: {evidence}
  See golden file: `{golden-file-path}:{line-range}`

### Completeness

**Verdict:** [PASS] / [FAIL]

- [ERROR] `{file}:{line}` -- {description} (missing_implementation)
  Evidence: {evidence}

### Correctness

**Verdict:** [PASS] / [FAIL]

- [ERROR] `{file}:{line}` -- {description}
  Evidence: {evidence}

### Eval Summary

| Criterion | Verdict | Errors | Warnings | Info |
|-----------|---------|--------|----------|------|
| Scope Compliance | {PASS/FAIL/SKIPPED} | {N} | {N} | {N} |
| Convention Adherence | {PASS/FAIL/SKIPPED} | {N} | {N} | {N} |
| Completeness | {PASS/FAIL/SKIPPED} | {N} | {N} | {N} |
| Correctness | {PASS/FAIL/SKIPPED} | {N} | {N} | {N} |

**Total findings:** {N} | **Errors:** {N} | **Warnings:** {N} | **Info:** {N}
```

### Skipped Criterion (D-04)

```
### {Criterion Name}

[SKIPPED] Disabled in config. Enable via `eval.criteria.{criterion_key}: true` in config.yml.
```

### User Gate -- Interactive Mode (GATE-01, D-07)

Findings grouped by criterion, severity-sorted (ERRORs first).

```
## Eval Gate

{N} finding(s) require your decision.

### Scope Compliance ({N} findings)

1. [ERROR] `{file}:{line}` -- {description}
   Evidence: {evidence}
   Action? [debug / ignore / defer]

2. [WARN] `{file}:{line}` -- {description}
   Evidence: {evidence}
   Action? [debug / ignore / defer]

### Convention Adherence ({N} findings)

3. [WARN] `{file}:{line}` -- Violates `{convention-name}`
   Evidence: {evidence}
   Action? [debug / ignore / defer]
```

Gate actions:
| Action | Label | Behavior |
|--------|-------|----------|
| debug | `debug` | Finding sent to debug agent for fix |
| ignore | `ignore` | Finding dismissed; pattern recorded in learnings.md as IGNORE entry (D-10) |
| defer | `defer` | Finding appended to learnings.md as TODO with file:line context (D-08) |

### User Gate -- Auto-Debug Mode (GATE-02)

```
## Eval Gate (auto-debug)

{N} finding(s) -- all sent to debug agent automatically.
```

### User Gate -- Auto-Skip-Minor Mode (GATE-03, D-09)

```
## Eval Gate (auto-skip-minor)

{N_skipped} INFO finding(s) auto-skipped.
{N_debug} WARN + ERROR finding(s) sent to debug agent automatically.
```

### Debug Cycle Section (appended to verify report per D-19)

```
## Debug Cycle {N}

**Cycle:** {N}/{max_cycles}
**Findings targeted:** {N}
**Fix plans:** {N} (grouping {N} findings by file)

### Fix Plan: {file-group-description}

**Files:** `{file1}`, `{file2}`
**Findings addressed:**
- [ERROR] `{file}:{line}` -- {description}
- [WARN] `{file}:{line}` -- {description}

**Result:** {fixed / partially fixed / failed}
**Commit:** `{short-hash}` -- {commit message}

### Re-Verify Results

**Files re-verified:** {N}
**New issues:** {N}

### Re-Eval Results

**Findings re-evaluated:** {N}
**Resolved:** {N}
**Remaining:** {N}
**New (from fix):** {N}
```

### Design Decision Escalation (DBUG-05, D-12, D-29)

```
## Design Decision Required

Finding: [ERROR] `{file}:{line}` -- {description}

This fix requires a design choice that changes {public API / functionality / scope}.

**Options:**

1. {Option A description}
   Impact: {what changes}

2. {Option B description}
   Impact: {what changes}

3. Skip this finding
   Impact: Finding remains unresolved

Which option? [1 / 2 / 3]
```

### Debug Cycle Max Reached (D-14, D-20)

```
## Debug Complete (max cycles reached)

**Cycles completed:** {N}/{max_cycles}
**Resolved:** {N} of {total} finding(s)
**Remaining:** {N} finding(s)

### Remaining Findings

- [ERROR] `{file}:{line}` -- {description}
  Attempted in cycle(s): {cycle-numbers}
  Suggested manual fix: {suggestion}

- [WARN] `{file}:{line}` -- {description}
  Attempted in cycle(s): {cycle-numbers}
  Suggested manual fix: {suggestion}

Action? [retry / ignore-all / defer-all]
```

### Debug Complete -- All Resolved

```
## Debug Complete

**Cycles used:** {N}/{max_cycles}
**All {total} finding(s) resolved.**

Proceeding to final summary.
```

### Progress Messages

| Element | Copy |
|---------|------|
| Eval start | `## Evaluating changes...` |
| Eval complete | `Eval complete: {N} findings ({errors} errors, {warnings} warnings, {info} info)` |
| Gate decision | `Gate: {N} to debug, {N} ignored, {N} deferred` |
| Debug cycle start | `## Debug cycle {N}/{max}...` |
| Fix plan start | `Fixing {N} finding(s) in {file-list}...` |
| Fix complete | `Fix applied: `{short-hash}` -- {N} finding(s) addressed` |
| Re-verify start | `Re-verifying {N} changed file(s)...` |
| Re-eval start | `Re-evaluating {N} targeted finding(s)...` |
| Cycle result | `Cycle {N}: {resolved}/{targeted} resolved, {new_findings} new` |
| Loop end (all clear) | `All findings resolved after {N} cycle(s).` |
| Loop end (max cycles) | `Max debug cycles ({max}) reached. {remaining} finding(s) remain.` |

### Empty States

| Element | Copy |
|---------|------|
| No findings | `[PASS] No findings. All criteria passed. Proceeding to summary.` |
| No enabled criteria | `[SKIPPED] All eval criteria disabled in config. Skipping evaluation.` |
| No verify results | `[ERROR] No verify results found. Run verification first. Eval requires verify output.` |
| No orient artifacts (MCP standalone, D-31) | `[WARN] Orient artifacts not found. Evaluating convention_adherence and correctness only. scope_compliance and completeness unavailable without scope contract.` |
| No findings to debug (after gate) | `No findings selected for debug. Proceeding to summary.` |
| No changed files after fix | `Debug fix produced no file changes. Finding may require manual intervention.` |

### Error States

| Element | Copy |
|---------|------|
| Eval LLM failure (D-26) | `[ERROR] Eval agent unavailable: {error_reason}. Retrying once...` |
| Eval LLM retry failed (D-26) | `[ERROR] Eval agent failed after retry: {error_reason}. Verify results remain valid. Skipping evaluation.` |
| Debug crash with commits (D-27) | `[ERROR] Debug agent crashed during cycle {N}. Committed fixes preserved (see git log). Uncommitted changes discarded. {N_fixed} of {N_targeted} findings were fixed before crash.` |
| Debug crash no commits (D-27) | `[ERROR] Debug agent crashed during cycle {N}. No fixes were committed. All findings remain unresolved.` |
| Report file missing | `[ERROR] Verify report not found at \`{path}\`. Cannot append eval results. Run verification first.` |
| Bootstrap required | `No bootstrap data found. Run /codescope:bootstrap first.` |
| Diff chunk too large (D-22) | `Large diff detected ({N} files). Evaluating in {chunk_count} chunk(s) of ~{threshold} tokens each.` |
| Finding parse failure | `[WARN] Could not parse {N} finding(s) from eval response. Raw response saved to report for manual review.` |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Ignore finding at gate | No confirmation -- action is non-destructive (pattern recorded in learnings.md for future filtering) |
| Defer to TODO at gate | No confirmation -- appends to learnings.md with context (D-08) |
| Debug fix commit (D-16) | No confirmation -- atomic commits per finding group, individually revertable via `git revert {hash}` |
| Retry after max cycles | No confirmation -- same cycle cap applies to retry |
| Ignore-all remaining | No confirmation -- findings recorded, user explicitly chose to dismiss |

## Ignore Pattern Format (D-10, GATE-04)

Recorded in `learnings.md` as structured IGNORE entries:

```markdown
### IGNORE: {description}

- **Status:** IGNORE
- **Pattern:** `{convention_name_or_finding_description}`
- **Scope:** `{file_glob_pattern}` (e.g., `tests/**`, `**/*.test.ts`, `*`)
- **Criterion:** `{criterion_name}` (e.g., `convention_adherence`)
- **Recorded:** {ISO timestamp}
- **Context:** Ignored at eval gate for task `{task_slug}`
```

---

## TODO Deferral Format (D-08, GATE-01)

Recorded in `learnings.md` as structured TODO entries:

```markdown
### TODO: {description}

- **Status:** TODO
- **File:** `{file}:{line}`
- **Severity:** {ERROR/WARN/INFO}
- **Criterion:** `{criterion_name}`
- **Evidence:** {evidence from finding}
- **Recorded:** {ISO timestamp}
- **Context:** Deferred at eval gate for task `{task_slug}`
```

## Agent Module Contract

All Phase 6 agent modules follow the established Options + Result + async function pattern (Phase 2 D-05):

```typescript
// Eval agent
interface EvalOptions {
  projectRoot: string;
  taskSlug: string;
  verifyResult: { static: StaticVerifyResult; runtime: RuntimeVerifyResult };
  scopeContractPath: string;
  planPath: string;
  coordinationPath: string;
  researchPath: string | null;
  enabledCriteria: Record<EvalCriterion, boolean>;
  ignorePatterns: IgnorePattern[];
}

interface EvalResult {
  criteria: EvalCriterionResult[];
  findings: EvalFinding[];
  overallStatus: "PASS" | "FAIL";
  timing_ms: number;
}

// Debug agent
interface DebugOptions {
  projectRoot: string;
  taskSlug: string;
  findings: EvalFinding[];
  scopeContractPath: string;
  reportPath: string;
  maxCycles: number;
  goldenFiles: Map<string, string>;
}

interface DebugResult {
  cyclesUsed: number;
  resolved: EvalFinding[];
  remaining: EvalFinding[];
  commits: Array<{ hash: string; findings: string[] }>;
  escalated: EvalFinding[];
  timing_ms: number;
}
```
