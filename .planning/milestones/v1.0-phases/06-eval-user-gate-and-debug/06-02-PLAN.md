    Create `src/debug/debug-agent.ts` following the agent module pattern:

    1. `runDebug(options: DebugOptions, callbacks: DebugCallbacks): Promise<DebugResult>`
       - Record start time
       - Initialize tracking: `resolved: EvalFinding[] = []`, `escalated: EvalFinding[] = []`, `commits: Array<{hash, findings}> = []`
       - Set `remaining = [...options.findings]`
       - Separate design decisions: `const designDecisions = remaining.filter(f => isDesignDecision(f))`
       - For each design decision:
         - Build DesignDecision object with 3 options: option A (fix approach), option B (alternative approach), option C (skip finding)
         - Call `callbacks.onDesignDecision(decision)` to get user choice
         - If user chose to skip -> move to escalated
         - If user chose an option -> add to fixable findings with the chosen approach noted in description
       - Set `fixable = remaining.filter(f => !isDesignDecision(f))`
       - Loop for `cycle = 1` to `options.maxCycles`:
         - If no fixable findings with severity ERROR or WARN remain: break (per D-20)
         - Call `callbacks.onProgress(\`## Debug cycle ${cycle}/${options.maxCycles}...\`)`
         - Create fix plan: `const plan = createFixPlan(fixable, options.taskSlug, options.scopeContractPath)`
         - For each agent in plan.agents:
           - Build fix prompt via `buildFixPrompt(...)` with findings and golden files
           - Dispatch via `callbacks.dispatchFixAgent(prompt)`
           - Track result (success/failure)
           - On success: extract commit hash from output if present
         - Scoped re-verify: call `callbacks.dispatchVerifyAgent(changedFiles)` where changedFiles comes from the fix plan's exclusiveWriteFiles
         - Scoped re-eval: call `callbacks.dispatchEvalAgent(scopedPrompt)` targeting only the findings from this cycle
         - Parse re-eval response to get remaining findings
         - Determine resolved: findings from this cycle that no longer appear in re-eval results
         - Move resolved to `resolved` array
         - Add new findings from re-eval to `fixable`
         - Build DebugCycleResult and call `appendDebugCycleSection(options.reportPath, cycle, cycleResult)`
         - Call `callbacks.onProgress(\`Cycle ${cycle}: ${resolved.length}/${fixable.length} resolved, ${newFindings.length} new\`)`
       - If loop ended by maxCycles:
         - Call `callbacks.onProgress(\`Max debug cycles (${options.maxCycles}) reached. ${remaining.length} finding(s) remain.\`)`
       - Else if all resolved:
         - Call `callbacks.onProgress(\`All findings resolved after ${cyclesUsed} cycle(s).\`)`
       - Return DebugResult with cyclesUsed, resolved, remaining (fixable that weren't resolved), commits, escalated, timing_ms
