    **src/execution/wave-scheduler.ts**: Implement wave-based execution scheduling per EXEC-07, with file overlap validation per EXEC-10.

    Import AgentAssignment and ExecutionWave types from src/orient/types.ts (or define locally if types module not yet available -- use the same field names and structure so it can be swapped to an import later).

    Export `buildWaveSchedule(agents: AgentAssignment[]): { waves: ExecutionWave[]; strategy: 'sequential' | 'parallel' | 'wave-based' }`:
    1. Build dependency DAG: for each agent, record its dependsOn list
    2. Topological sort into waves:
       - Wave 1: agents with empty dependsOn
       - Wave N: agents whose ALL dependencies are in waves < N
       - Detect cycles: if any agent can't be placed, throw Error(`Circular dependency detected involving: ${unplaced.join(', ')}`)
    3. Within each wave, check file overlap per EXEC-10:
       - For each pair of agents in the wave, compute set intersection of exclusiveWriteFiles
       - If overlap found: split the overlapping agents into separate sub-waves (increment wave numbers)
       - Agents without overlap remain in the same wave as 'parallel'
       - Single-agent waves always mode 'sequential'
       - Multi-agent waves with no file overlap: mode 'parallel'
    4. Determine strategy:
       - All waves single-agent -> 'sequential'
       - Single wave with multiple agents -> 'parallel'
       - Multiple waves with at least one multi-agent -> 'wave-based'
    5. Return waves array (sorted by waveNumber ascending) and strategy

    Export `validateFileOverlap(agents: AgentAssignment[], waves: ExecutionWave[]): { name: string; status: 'PASS' | 'FAIL'; detail?: string }[]`:
    - Per D-19 check (1): no overlapping file writes in same parallel wave
    - For each wave where mode is 'parallel':
      - Build map of agent name -> Set<exclusiveWriteFiles>
      - Check all agent pairs for set intersection
      - If overlap: return FAIL check with detail: `{agentA} and {agentB} both write: {overlapping files comma-separated}`
    - If no overlaps found: return single PASS check

    Export `validateDependencyOrdering(agents: AgentAssignment[], waves: ExecutionWave[]): { name: string; status: 'PASS' | 'FAIL'; detail?: string }[]`:
    - Per D-19 check (2): no agent depends on an agent in the same wave, no circular dependencies
    - Build wave lookup: agentName -> waveNumber
    - For each agent with dependsOn entries, verify all dependencies are in earlier waves
    - Check for circular dependencies (should already be caught by buildWaveSchedule, but belt-and-suspenders)

    Export `validateScopeCoverage(agents: AgentAssignment[], inScopeItems: string[]): { name: string; status: 'PASS' | 'FAIL' | 'WARNING'; detail?: string }[]`:
    - Per D-19 check (3): every In Scope item has at least one agent assigned
    - Build set of all agent task descriptions + file assignments
    - For each inScopeItem, check if any agent's task or files reference it
    - Return PASS if all covered, WARNING with uncovered items if not

    **Tests**: Create test plans with known agent configurations. Test all edge cases: no dependencies (all parallel), linear chain (all sequential), diamond dependencies, file overlaps triggering sub-wave splits, cycle detection.
