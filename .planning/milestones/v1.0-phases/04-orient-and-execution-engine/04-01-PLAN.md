    **src/orient/analysis.ts**: Implement graph-based analysis for a task's affected scope.

    Export `AnalysisOptions` interface: `{ projectRoot: string; taskSlug: string; keywords: string[]; outputDir: string }`

    Export `runAnalysis(options: AnalysisOptions): Promise<AnalysisResult>`:
    1. Start timer
    2. Call `getGraph(options.projectRoot)` to get cached graph + centralities
    3. **Affected files** (per ORNT-08):
       - Walk all graph nodes, match keywords against node name (lowercase) and filePath (lowercase)
       - For each match, create AffectedFile with: filePath from node attrs, risk = classifyRisk(centrality) using the same thresholds as orient.ts (>0.7 HIGH, >=0.3 MEDIUM, else LOW), centrality from centralities map, community from node attrs community field
       - Sort by centrality descending, cap at 50 files
    4. **Blast radius** (per ORNT-08):
       - Take top 5 highest-centrality affected nodes
       - For each, call `blastRadius(graph, nodeId, 3)` from analytics.ts
       - Collect all unique filePaths across all blast radius results
       - Store as `{ filePath, hopDistance, riskLevel }` using the hop-distance classification from blastRadius return
    5. **Convention matching**:
       - Get codescopePath via `getCodescopePath(projectRoot)`
       - Extract filePaths from affectedFiles
       - Call readRelevantConventions logic (re-implement the parsing from orient.ts or import if extracted to shared util) to get conventions matching affected files
       - Store convention names in conventionMatches array
    6. **Test file mapping**:
       - For each affected node, check graph neighbors (outEdges + inEdges) for nodes where name includes '.test.' or '.spec.' or kind is 'file' and filePath contains '/test' or '/tests/'
       - Collect unique test file paths
    7. **Cross-community impact**:
       - Group affected nodes by community ID
       - Count total nodes in each community (from full graph)
       - Return array of `{ communityId, nodeCount (total in community), affectedCount (affected in this task) }`
    8. Return AnalysisResult with durationMs

    Export `writeAnalysisArtifact(result: AnalysisResult, outputDir: string): string`:
    - Write analysis.md to outputDir with:
      - H1 "Analysis" with completion timestamp
      - H2 "Affected Files" table (filePath, risk, centrality, community)
      - H2 "Blast Radius" table (filePath, hopDistance, riskLevel)
      - H2 "Conventions in Scope" list
      - H2 "Test Files" list
      - H2 "Cross-Community Impact" table
    - Return written file path

    **readRelevantConventions helper**: Either extract from orient.ts into a shared utility (preferred) or re-implement the same parsing logic inline. The function reads conventions.md, splits by `## ` headings, parses `**Convention:**` and `**Files:**` fields, and returns convention names where files overlap with the given file paths. Use the exact same parsing pattern as src/tools/orient.ts lines 89-129.
