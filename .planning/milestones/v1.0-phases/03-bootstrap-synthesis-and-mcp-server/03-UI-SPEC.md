# Phase 3 -- UI Design Contract

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase type: Backend / CLI.** This phase has no browser-based frontend. The interaction surface is Claude Code's terminal (markdown-rendered skill output) and MCP tool JSON responses consumed by AI agents. The contract below specifies the **output formatting, structured data shapes, and copywriting** that constitute this phase's "UI."

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | not applicable (no frontend) |
| Icon library | not applicable (terminal output uses Unicode characters only) |
| Font | Terminal default (monospace, controlled by user's terminal emulator) |

**Rationale:** CodeScope Phase 3 is a Claude Code plugin. All output renders as markdown in the Claude Code terminal or as structured JSON consumed by MCP tool callers. There are no browser components, CSS, or visual design tokens to specify. The contract below governs structured output formatting instead.

### Bootstrap --force Confirmation (D-30)

| Element | Copy |
|---------|------|
| Heading | `## Re-bootstrap Confirmation` |

| Preserved note | `Preserved (not affected): config.yml, conventions-enforced.md, learnings.md, orient/, plans/, execution/, reports/` |
| Prompt | `Proceed with full re-bootstrap? [Y/n]` |

### MCP Tool Response Contract (D-17, D-18, D-19)

**Success response:**
```json
{
  "status": "ok",
  "data": { "..." },
  "metadata": {
    "last_bootstrap": "2026-03-23T10:00:00Z",
    "staleness": "fresh",
    "query_ms": 42
  }
}
```

**Error response:**
```json
{
  "status": "error",
  "error": {
    "code": "NOT_BOOTSTRAPPED",
    "message": "No bootstrap data found. Run /codescope:bootstrap first.",
    "recovery": "Run /codescope:bootstrap to analyze your codebase."
  }
}
```

**Partial response:**
```json
{
  "status": "partial",
  "data": { "..." },
  "warnings": [
    "Convention detection skipped for 2 services (exceeded squad cap)"
  ],
  "metadata": { "..." }
}
```

### Staleness Labels (D-18)

| Staleness | Threshold | Display |
|-----------|-----------|---------|
| fresh | <7 days since last bootstrap | No warning |
| stale | 7-30 days | `"staleness": "stale"` + metadata note: `"Consider running /codescope:bootstrap to refresh analysis."` |
| very_stale | >30 days | `"staleness": "very_stale"` + metadata note: `"Analysis is outdated. Run /codescope:bootstrap --force for accurate results."` |

### Partial Tool Capabilities (D-36, D-37, D-38)

Tools with partial Phase 3 functionality include capability metadata:
```json
{
  "metadata": {
    "capabilities": ["convention_compliance"],
    "upcoming": ["blast_radius_diff", "build_verification", "test_verification"]
  }
}
```

### MCP Tool Error Codes and Recovery Copy

| Error Code | Message | Recovery |
|------------|---------|----------|
| NOT_BOOTSTRAPPED | No bootstrap data found. | Run /codescope:bootstrap to analyze your codebase. |
| STALE_DATA | Bootstrap data is older than 30 days. | Run /codescope:bootstrap to refresh analysis. |
| INVALID_INPUT | {Zod validation message} | Check parameter types and required fields. See tool description for schema. |
| NODE_NOT_FOUND | Symbol "{name}" not found in the knowledge graph. | Verify the symbol name and file path. Run codescope_search to find similar symbols. |
| SERVICE_NOT_FOUND | Service "{name}" not found. | Run codescope_service_map to list available services. |
| GRAPH_LOAD_FAILED | Failed to load knowledge graph from database. | Database may be corrupted. Run /codescope:bootstrap --force to rebuild. |
| TIMEOUT | Graph query exceeded time limit. | Try a more specific query or reduce hop depth. |
| PARTIAL_ANALYSIS | {N} services exceeded squad cap and received lightweight analysis only. | Increase bootstrap.squad_cap in config.yml and re-bootstrap. |

### Empty State Copy

| Context | Copy |
|---------|------|
| No bootstrap data (any MCP tool) | "No bootstrap data found. Run /codescope:bootstrap to analyze your codebase." |
| No conventions detected | "No conventions detected. This may indicate a very small codebase or highly varied coding patterns." |
| No danger zones | "No high-risk files identified. All files have low centrality scores." |
| No cross-service dependencies | "No cross-service dependencies detected. Services appear to be independent." |
| Empty search results | "No symbols matching \"{query}\" found in the knowledge graph." |
| No enforced conventions | "No conventions enforced yet. Use /codescope:review-learnings (Phase 7) to promote high-confidence conventions." |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| bootstrap --force | Brief confirmation listing what will be rebuilt vs. preserved. User confirms Y/n. (D-30) |

No other destructive actions exist in Phase 3. Convention enforcement promotion is deferred to Phase 7.

## Structured Output Contracts

> This section replaces the traditional component inventory for this backend/CLI phase.

### Readiness Score Format (D-01, D-02, D-03, D-04, D-05)

The readiness.md artifact follows this structure:

```markdown
# AI Readiness Score: {grade} ({percent}%)

## Overall Assessment
{one-sentence summary of what this grade means for AI-assisted development}

## Dimension Scores

| Dimension | Score | Grade | Delta | What This Means for AI |
|-----------|-------|-------|-------|------------------------|
| Convention Coverage | {pct}% | {grade} | {+/-N%} | {D-04 explainer} |
| Type Safety | {pct}% | {grade} | {+/-N%} | {D-04 explainer} |
| Test Coverage Proxy | {pct}% | {grade} | {+/-N%} | {D-04 explainer} |
| Import Graph Health | {pct}% | {grade} | {+/-N%} | {D-04 explainer} |

## Top 3 Improvements
1. {specific action} -- {file/pattern reference}
2. {specific action} -- {file/pattern reference}
3. {specific action} -- {file/pattern reference}

## Grading Scale
A+: 97-100% | A: 93-96% | A-: 90-92%
B+: 87-89% | B: 83-86% | B-: 80-82%
C+: 77-79% | C: 73-76% | C-: 70-72%
D+: 67-69% | D: 63-66% | D-: 60-62%
F: <60%
```

### Cross-Service Map Format (D-10, D-11, D-33)

The cross-service-map.md artifact follows this structure:

```markdown
# Cross-Service Dependency Map

## Services

| Service | Path | LOC | Framework | Analysis |
|---------|------|-----|-----------|----------|
| {name} | {path} | {loc} | {framework} | Full / Lightweight |

## Dependencies

| From | To | Shared Types | Import Count |
|------|-----|-------------|--------------|
| {service-a} | {service-b} | {type1, type2} | {N} |

## Merged Conventions
| Convention | {service-a} | {service-b} | {shared} |
|------------|-------------|-------------|----------|
| {pattern} | {pct}% | {pct}% | {pct}% |
```

### Incremental Bootstrap Behavior (D-09)

| Scenario | Behavior | Output |
|----------|----------|--------|
| <50% files changed | Incremental: re-analyze only changed services/files | `## Incremental bootstrap ({N} files changed)` |
| >=50% files changed | Auto-fallback to full | `## Full bootstrap (>{pct}% files changed, exceeds incremental threshold)` |
| --force flag | Full re-bootstrap | `## Full bootstrap (--force)` |
| First run | Full bootstrap | `## Bootstrap` |
