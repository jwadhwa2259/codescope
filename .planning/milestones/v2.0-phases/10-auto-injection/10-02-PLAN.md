<objective>
Create the hook scripts that read pre-computed artifact files and inject context into Claude's reasoning on every Edit/Write operation. PreToolUse delivers proactive warnings (danger zones, conventions, blast radius). PostToolUse reminds Claude of conventions and blast radius for the just-modified file. Both stay within 500-token budget via priority queue. Both silently no-op when bootstrap data is missing.

Purpose: This is the user-facing deliverable of Phase 10 -- invisible codebase intelligence on every file edit. Claude sees context without the user asking for it. Combined with Plan 01's artifact generation, this creates a complete auto-injection pipeline (per D-01 through D-15).

Output: Hook scripts in `src/hooks/`, hooks registration in `hooks/hooks.json`, build config update, plugin manifest update, full test coverage.

    - Import only `readFileSync`, `existsSync` from `"node:fs"` and `join` from `"node:path"`

    - Export `function readJsonSafe<T>(filePath: string): T | null`:
      - `if (!existsSync(filePath)) return null;`

    - Export `interface ArtifactData`:

      }

    **src/hooks/lib/budget-composer.ts** -- Priority-budgeted token composition per D-04:
    - Export `const MAX_TOKENS = 500;`
    - Export `function estimateTokens(text: string): number` -- `Math.ceil(text.length / 4)` (returns 0 for empty string)
    - Export `interface InjectionItem { priority: number; content: string; }` where priority 1=danger zone, 2=conventions, 3=blast radius, 4=general
    - Export `function composeBudgetedMessage(items: InjectionItem[], maxTokens: number = MAX_TOKENS): string`:
      - Sort items by priority ascending (1 first)
      - Iterate sorted items, accumulate tokensUsed, add item.content to parts array if tokensUsed + itemTokens <= maxTokens
      - Return `parts.join("\n\n")` (empty string if no items)

    **src/hooks/pre-tool-use.ts** -- PreToolUse hook entry point (INJECT-01, INJECT-03, INJECT-04, INJECT-05):
    - Import ONLY from `"node:fs"`, `"node:path"`, and `"./lib/*.js"`
    - Export `function processPreToolUse(input: HookInput, projectDir: string): PreToolUseOutput` for testability
    - At module level (for CLI execution): read stdin via `readFileSync(0, "utf-8")` (cross-platform per Pitfall 3), parse JSON, call processPreToolUse, write JSON to stdout, exit 0
    - Guard the stdin reading with `if (process.argv[1] && ...import.meta.url...)` or similar so the module can be imported for testing without executing
    - `processPreToolUse` logic:
      1. `const codescopeDir = join(projectDir, ".claude", "codescope")`
      2. Guard: if `!existsSync(join(codescopeDir, "graph.db"))` -> return `{ hookSpecificOutput: { hookEventName: "PreToolUse" } }` (D-14)
      3. Normalize file path: `const rawPath = input.tool_input.file_path; const absPath = isAbsolute(rawPath) ? rawPath : resolve(input.cwd, rawPath); const relPath = relative(projectDir, absPath).split("\\").join("/")`
      4. Read artifacts: `const artifacts = readAllArtifacts(join(codescopeDir, "injection"))`
      5. Check trigger threshold (D-08):
         - `const dzEntry = artifacts.dangerZones?.files[relPath]`
         - `const convEntries = artifacts.conventions?.files[relPath]`
         - `const centrality = dzEntry?.centrality ?? 0`
         - `const hasConventions = convEntries && convEntries.length > 0`
         - If `centrality <= 0.3 && !hasConventions` -> return bare output (no additionalContext)
      6. Build injection items (D-04 priority queue):
         - Priority 1 (danger zone): if dzEntry exists and riskScore > 0.1, format as `[DANGER ZONE] {relPath} (risk: {riskScore})` + indented reasons
         - Priority 2 (conventions): if convEntries exist, sort by confidence (HIGH-CONF first using order map `{ "HIGH-CONF": 0, "MEDIUM-CONF": 1, "LOW-CONF": 2 }`), format as `[CONVENTIONS]` header + indented bullet per convention: `- {name} ({adoption_pct}% adoption, {confidence})`
         - Priority 3 (blast radius): `const brEntry = artifacts.blastRadius?.files[relPath]`, if exists and totalAffected > 1, format as `[BLAST RADIUS] {totalAffected} files affected` + risk breakdown line + top 3 dependents
      7. Compose: `const message = composeBudgetedMessage(items)`
      8. Return: `{ hookSpecificOutput: { hookEventName: "PreToolUse", ...(message ? { additionalContext: message } : {}) } }`
