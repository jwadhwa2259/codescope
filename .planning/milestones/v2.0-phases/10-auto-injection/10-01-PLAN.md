<objective>
Build the artifact generation pipeline that produces pre-computed JSON index files for hook scripts to read. Three index files (danger-zones.json, conventions.json, blast-radius.json) are written to `.claude/codescope/injection/` after every bootstrap and every incremental rebuild. All heavy computation (graph traversal, centrality, community detection) happens here in the MCP server process -- hook scripts will only read the resulting JSON.

Purpose: Hooks need fast O(1) file path lookups into pre-computed data. This plan creates the data layer that Plan 02's hook scripts consume. Decouples expensive graph computation from the ~50ms hook budget (per D-01, D-02, D-11, D-12, D-13).

Output: `src/artifacts/` module with types + generators + integration into bootstrap and incremental rebuild pipelines.
</objective>

      files: Record<string, DangerZoneFileEntry>;
    }

    // ConventionIndex -- keyed by relative file path
    export interface ConventionFileEntry {
      name: string;

    }

    }

      byRisk: { red: number; orange: number; yellow: number; green: number };

    }
    export interface BlastRadiusIndex {

    }

    **src/artifacts/blast-radius-index.ts** -- Build blast radius index from graph:
    - Export `function buildBlastRadiusIndex(db: DatabaseType): BlastRadiusIndex`
    - Load graph via `loadGraphFromSQLite(db)`, compute centrality

    - Classify results into byRisk counts: red/orange/yellow/green
    - Extract topAffected: first 5 unique file paths from blast radius results (excluding the file itself), sorted by hop ascending
    - Skip file nodes with centrality <= 0.1 for performance (they contribute nothing meaningful to blast radius)
    - Return `{ generated: new Date().toISOString(), files }` typed as `BlastRadiusIndex`
