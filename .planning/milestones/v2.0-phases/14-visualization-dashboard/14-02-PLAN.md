**Create `src/dashboard/client/lib/api-client.ts`:**

Typed fetch wrapper for all dashboard API endpoints. Class or factory function pattern:

```typescript
export interface ApiClient {
  fetchGraph(): Promise<GraphApiResponse>;
  fetchConventions(): Promise<ConventionsApiResponse>;
  fetchReadiness(): Promise<ReadinessApiResponse>;
  fetchBlastRadius(file: string, direction?: 'forward' | 'reverse'): Promise<BlastRadiusApiResponse>;
  fetchStatus(): Promise<StatusApiResponse>;
  postReview(filePaths: string[], branch?: string): Promise<any>;
  postImpact(filePaths: string[], maxHops?: number): Promise<any>;
}
```

- Base URL: `window.location.origin` (same host)
- All GET requests use `fetch(url).then(r => r.json())`
- POST requests use `fetch(url, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(data) })`
- Error handling: if `response.status >= 400`, throw with parsed error body
- The `fetchBlastRadius` method URL-encodes the file path: `/api/blast-radius/${encodeURIComponent(file)}?direction=${direction}`
- Export `createApiClient()` factory function

**Create `src/dashboard/client/lib/ws-client.ts`:**

WebSocket connection manager with auto-reconnect and exponential backoff (per D-34).

```typescript
export interface WebSocketClient {
  connect(): void;
  disconnect(): void;
  onEvent(handler: (event: WSEvent) => void): () => void;  // returns unsubscribe
  isConnected(): boolean;
}
```

- URL: `ws://${window.location.host}/ws`
- On open: set connected flag, call `onStatusChange('connected')` callback, reset backoff
- On close: set disconnected, call `onStatusChange('disconnected')`, schedule reconnect
- Reconnect backoff: 1s, 2s, 4s, 8s, 16s, max 30s (per D-34)
- On message: parse JSON, dispatch to registered handlers via `Set<(event: WSEvent) => void>`
- On reconnect: emit `'reconnected'` event so app can refresh all data
- Export `createWebSocketClient(onStatusChange)` factory
