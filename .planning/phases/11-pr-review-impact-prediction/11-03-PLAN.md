    Follow the exact same file structure as `skills/settings/SKILL.md` -- YAML frontmatter delimited by triple-dash lines, then markdown body.

    Frontmatter fields:
    - name: review
    - description: Run structural impact analysis on a branch, PR, or working tree diff.
    - allowed-tools: [Bash, Read, mcp__codescope__codescope_review]

    Body starts with `# /codescope:review` heading.

    Body content instructs Claude to:

    1. **Parse $ARGUMENTS** to determine review target:
       - If argument looks like a number (e.g., `123`, `#456`): treat as PR number, strip `#` prefix
       - If argument looks like a branch name (e.g., `feature/auth`, `fix-bug`): treat as branch
       - If no arguments or empty: use working tree diff (default)

    2. **Call the codescope_review MCP tool** with the appropriate parameter:
       - PR number: `{ "pr_number": <number> }`
       - Branch: `{ "branch": "<branch-name>" }`
       - Working tree: `{}` (no parameters)

    3. **Format the response as markdown** using this template structure:
       - H2 heading: "Code Review: {source description}"
       - Metadata line: Reviewed timestamp, Source (PR/branch/working tree), Files analyzed count
       - H3 "Risk Summary" -- table with HIGH/MEDIUM/LOW counts from summary
       - H3 "File Analysis" -- table with columns: File, Risk, Centrality, Blast Radius
       - H3 "Dependency Changes" (only if new_edges or removed_edges exist) -- list edges with source->target(kind) format
       - "Circular Dependencies Detected" subsection (only if circular_dependencies non-empty) -- list cycle paths
       - H3 "Convention Violations" (only if convention_violations non-empty) -- for each: bold convention name with confidence and adoption_pct, file, evidence
       - H3 "Cross-Community Warning" (only if cross_community_changes.flagged is true) -- show communities_touched count, threshold note, and per-community breakdown with label and files
       - H3 "Metadata" -- last bootstrap, staleness, query time

    4. **Error handling:**
       - NOT_BOOTSTRAPPED: "CodeScope has not analyzed this project yet. Run `/codescope:bootstrap` first."
       - GH_CLI_UNAVAILABLE: Display error message + suggest using branch name: "The `gh` CLI is not available. Try `/codescope:review feature-branch` instead."
