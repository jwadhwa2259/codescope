    **src/execution/orchestrator.ts**: Implement the thin execution orchestrator per EXEC-01, EXEC-05, EXEC-06.

    Export `runExecution(options: ExecutionOptions): Promise<ExecutionResult>`:

    The orchestrator is THIN (<15K tokens per EXEC-06). It:
    1. **Reads plan** from `options.planPath` (fs.readFileSync + parse into ExecutionPlan)
    2. **Initializes coordination** via `initCoordinationFile(executionDir, taskSlug, mode)`
    3. **Detects agent teams** via `detectAgentTeams()` to determine if parallel waves are possible
    4. **Builds wave schedule** from plan agents (may already be computed, verify with `buildWaveSchedule()`)
    5. **Executes waves sequentially** (wave 1, then wave 2, etc.):

       For each wave:
       a. Emit progress per D-24: `"Executing wave {N}/{total}: [{agent names}]..."`
       b. If wave mode is 'parallel' AND agent teams available:
          - Dispatch all wave agents concurrently (construct prompts + invocations)
          - Wait for all to complete
          - Handle failures individually
       c. If wave mode is 'sequential' OR agent teams unavailable:
          - Dispatch agents one at a time
          - Each agent:
            - Build prompt via `buildAgentPrompt(assignment, context)`
            - Build invocation via `buildAgentInvocation(assignment, prompt)`
            - Append 'started' coordination entry
            - Execute (NOTE: actual sub-agent dispatch via Task tool happens in the skill body -- the orchestrator prepares the invocation and the skill body calls the Tool)
            - On success: append 'done' coordination entry, write change report
            - On failure: append 'failed' entry, retry ONCE per D-36 with same context + error
            - On retry failure: skip agent + all dependents per D-37
       d. After each agent completes, emit progress per D-24: `"{agent} complete ({duration}s, {N} files)"`

    6. **Handles failures** per D-36/D-37/D-38:
       - Track failed agents and their dependents
       - Before dispatching an agent, check if any of its dependsOn agents failed -> skip with 'skipped' coordination entry
       - Independent agents (no shared dependencies with failed agent) continue per D-36

    7. **Writes execution summary** to `execution/[task-slug]/summary.md` per D-27:
       - Agent results table: name, status, duration, files, lines changed
       - Totals: succeeded/total, files changed, lines changed, tokens estimate, mode
       - Next step: "Proceeding to verification..." or partial results message
       - Failures section with error details, retry info, skipped dependents, suggested action per D-38

    8. **Returns ExecutionResult** with all agent results, summary path, coordination path

    Export `readPlanFromDisk(planPath: string): ExecutionPlan`:

    - Return structured plan
