# Phase 4 -- UI Design Contract

> Output format and interaction contracts for the orient and execution pipeline. This phase has NO browser UI -- all output is structured markdown artifacts, CLI progress messages, and MCP JSON responses rendered in Claude Code's terminal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | not applicable (CLI plugin) |
| Icon library | not applicable (text-only output) |
| Font | Terminal default (monospace, user-controlled) |

**Rationale:** CodeScope is a Claude Code plugin. All output renders in the user's terminal via Claude Code's chat interface. There are zero browser-rendered components. The "design system" is structured markdown with consistent heading levels, table formats, and progress message patterns established in Phases 1-3.

---

## Spacing Scale

Terminal output uses blank lines and markdown heading hierarchy for visual separation. No pixel-based spacing applies.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 blank lines | Consecutive related lines within a section |
| tight | 1 blank line | Between items in a list or table rows (markdown default) |
| section | 1 blank line + H2 heading | Between major sections in artifacts |
| subsection | 1 blank line + H3 heading | Between subsections within a major section |
| block | 2 blank lines | Between top-level document sections (frontmatter to body, body to footer) |

Exceptions: none

---

## Typography

Terminal rendering is monospace at user-controlled size. The contract governs **markdown heading levels** and **text formatting** as the typography equivalent.

| Role | Markdown | Formatting Rule |
|------|----------|-----------------|
| Document title | `# H1` | Exactly one per artifact. Document name only. |
| Major section | `## H2` | Pipeline phase names, gate labels, summary sections. Max 8 per artifact. |
| Subsection | `### H3` | Agent names, validation check groups, detail breakdowns. |
| Body text | Plain paragraph | No indentation. Max 80 characters per line for readability in narrow terminals. Wrap at sentence boundaries. |
| Emphasis | `**bold**` | Reserved for: status labels (PASS/FAIL/SKIP), risk levels (HIGH/MEDIUM/LOW), gate actions (APPROVED/REJECTED/EDITED). Never for decoration. |
| Code references | `` `backtick` `` | File paths, function names, config keys, CLI flags. Always use backticks. |
| Lists | `- ` unordered | Scope items, file lists, convention lists. No numbered lists except for ordered steps. |

---

## Color

Terminal color is not controlled by this plugin -- Claude Code renders markdown natively. The contract governs **semantic text markers** that convey the equivalent of color meaning.

| Role | Marker | Usage |
|------|--------|-------|
| Success | `PASS` or `complete` | Validation checks passed, agent completed, gate approved |
| Warning | `WARNING` or `escalated` | Structural issues at Gate 2, auto-fix applied, partial results |
| Error | `FAIL` or `failed` | Validation failed, agent crashed, gate rejected |
| Info | `SKIP` or `skipped` | Agent skipped due to dependency failure, low-impact research topic skipped |
| Risk HIGH | `**HIGH**` | Danger zone files, high-centrality nodes in scope |
| Risk MEDIUM | `**MEDIUM**` | Files with moderate centrality or convention conflicts |
| Risk LOW | `**LOW**` | Leaf files with few dependents |

Accent reserved for: status markers only (PASS/FAIL/WARNING/SKIP). Never used for decorative emphasis.

## Copywriting Contract

### Primary CTA (Skill Invocation)

| Element | Copy |
|---------|------|
| Skill trigger | `/codescope:orient [task description]` |
| Orient with no-confirm | `/codescope:orient [task] --no-confirm` |
| Orient with no-clarify | `/codescope:orient [task] --no-clarify` |

### Gate 1: Scope Contract Approval

| Element | Copy |
|---------|------|
| Gate heading | `## Scope Contract` |
| Approve prompt | `Approve this scope? [approve / edit / reject]` |
| Approved label | `**APPROVED** -- proceeding to research and planning.` |
| Rejected label | `**REJECTED** -- returning to clarification with your feedback.` |
| Edited label | `**EDITED** -- revised scope contract below.` |

### Gate 2: Plan Approval

| Element | Copy |
|---------|------|
| Gate heading | `## Execution Plan` |
| Approve prompt | `Approve this plan? [approve / edit / reject]` |
| Approved label | `**APPROVED** -- beginning execution.` |
| Rejected label | `**REJECTED** -- returning to scope contract.` |
| Edited label | `**EDITED** -- re-validating modified plan.` |

### Progress Messages (Brief Verbosity, per D-24)

| Element | Copy |
|---------|------|
| Phase banner | `## Clarifying scope...` / `## Researching...` / `## Planning...` / `## Validating plan...` / `## Executing...` |
| Wave start | `Executing wave {N}/{total}: [{agent-a}, {agent-b}]...` |
| Agent complete | `{agent-name} complete ({duration}s, {N} files)` |
| Agent failed | `{agent-name} **failed** ({error-reason}) -- retrying once...` |
| Agent retry failed | `{agent-name} **failed** after retry -- skipping + {N} dependents` |
| Execution summary | `## Summary\n\nTotal: {duration}s | Files changed: {N} | Agents: {succeeded}/{total} | Mode: {sequential/parallel/wave-based} | Tokens: ~{estimate}\n\nNext: {next-step}` |

### Progress Messages (Detailed Verbosity, per D-25)

Detailed mode adds to brief mode -- never replaces it.

| Element | Copy |
|---------|------|
| Agent file list | `  {agent-name}: writing {file-path}` |
| Agent change summary | `  {agent-name}: +{added}/-{removed} lines across {N} files` |
| Coordination entry | `  [{timestamp}] {agent-name}: {status} -- {detail}` |

### Empty States

| Element | Copy |
|---------|------|
| No bootstrap data | `CodeScope has not analyzed this codebase yet. Run \`/codescope:bootstrap\` first to build the knowledge graph.` |
| No clarification needed | `Task is specific enough -- skipping clarification. Proceeding to research.` |
| No research needed | `All affected libraries are internal -- skipping external research. Proceeding to planning.` |
| No conventions for scope | `No enforced conventions affect files in scope. Agents will follow golden file patterns.` |
| Empty execution (plan rejected) | `Plan rejected. No changes made. Run \`/codescope:orient\` again to restart.` |

### Error States

| Element | Copy |
|---------|------|
| Bootstrap stale | `WARNING: Bootstrap data is {N} days old. Results may not reflect recent changes. Consider running \`/codescope:bootstrap\` to refresh.` |
| Graph unavailable | `ERROR: Knowledge graph not found at \`{path}\`. Run \`/codescope:bootstrap\` first.` |
| Agent teams unavailable | `Agent teams not available -- running sequentially. Enable for faster parallel execution: set \`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=true\` in your environment.` |
| Agent teams hint (one-time) | `Tip: Enable agent teams for parallel execution. See \`/codescope:settings\` for details.` |
| Plan validation failed | `Plan validation failed: {specific-reason}. Auto-fixing... ({attempt}/{max_attempts})` |
| Plan validation unfixable | `WARNING: Could not auto-fix plan issue: {reason}. Flagged for your review at Gate 2.` |
| Partial execution | `{N}/{total} agents completed successfully. {M} failed, {K} skipped.\n\nPartial results are in your working tree as uncommitted changes. Review with \`git diff\` and decide what to keep.\n\nTo retry failed tasks: \`/codescope:orient\` with the same task description.` |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Scope rejection at Gate 1 | No confirmation needed -- returns to clarification (non-destructive) |
| Plan rejection at Gate 2 | No confirmation needed -- returns to scope (non-destructive, plan persisted to disk per D-16) |
| `--no-confirm` flag | No runtime confirmation -- user opted in via explicit flag. Scope and plan still written to disk for audit. |
| Overwriting existing execution directory | Auto-handled: new task-slug directory per execution. No overwrite risk. |

## Artifact Format Contracts

These are the structured markdown files produced by Phase 4. Each format is a binding contract -- downstream agents (verify, eval, debug) parse these programmatically.

### Scope Contract (`execution/[task-slug]/scope-contract.md`)

```markdown
# Scope Contract: {task-slug}

**Task:** {original user task description}
**Created:** {ISO timestamp}
**Status:** {APPROVED / REJECTED / PENDING}

## In Scope

- {specific deliverable 1}
- {specific deliverable 2}
- ...

## Out of Scope

- {explicitly excluded item 1}
- {explicitly excluded item 2}
- ...

## Affected Files (estimated)

| File | Risk | Centrality | Community |
|------|------|------------|-----------|
| `{path}` | **{HIGH/MEDIUM/LOW}** | {score} | {community-name} |

## Assumptions

- {assumption made when clarification was minimal}

## Conventions in Scope

- {convention name}: {brief description} (from `conventions-enforced.md`)

## Risk Flags

- {danger zone file in scope}: {why it is risky}
```

### Research Output (`execution/[task-slug]/research.md`)

```markdown
# Research: {task-slug}

**Completed:** {ISO timestamp}
**Topics researched:** {N}
**Topics skipped (low-impact):** {N}

## Relevant APIs

### {library/framework name}

{Code examples from Context7, version-specific notes}

## Best Practices

- {practice relevant to the task}

## Known Issues / Pitfalls

- {issue that affects this task}

## Version-Specific Notes

- {version constraint or breaking change}

## Skipped Topics

- {topic}: low graph impact ({centrality score}, {N} files using it)
```

### Execution Plan (`plans/[task-slug].md`)

```markdown
# Execution Plan: {task-slug}

**Created:** {ISO timestamp}
**Status:** {APPROVED / REJECTED / EDITED / PENDING}
**Strategy:** {sequential / parallel / wave-based}
**Estimated agents:** {N}
**Estimated total tokens:** ~{N}K

## Agents

### Agent: {agent-name}

- **Wave:** {N} (or "sequential: after {dependency-name}")
- **Task:** {what this agent does}
- **Files (exclusive write):** `{path1}`, `{path2}`
- **Files (read-only):** `{path3}`, `{path4}`
- **Conventions:** {list of enforced conventions for these files}
- **Golden files:** `{golden-file-path}` (lines {N}-{M} for key pattern)
- **Depends on:** {agent-name} or "none"
- **Estimated tokens:** ~{N}K
- **Timeout:** {N}s

## Execution Order

| Wave | Agents | Mode | Files |
|------|--------|------|-------|
| 1 | {agent-a}, {agent-b} | parallel (agent teams) | {file-count} |
| 2 | {agent-c} | sequential (depends on wave 1) | {file-count} |

## Validation

- [x] No overlapping file writes within waves: **PASS**
- [x] Dependency ordering (no cycles, no same-wave deps): **PASS**
- [x] Scope coverage (every In Scope item assigned): **PASS**
- [ ] {auto-fixed item}: **AUTO-FIXED** -- {detail}
- [ ] {escalated item}: **WARNING** -- {detail}

## Removed by User

{empty unless user edits plan at Gate 2}
```

## MCP Response Contracts

Orient pipeline MCP interactions follow the established D-17/D-18/D-19 response envelope from Phase 3.

| Response Type | Envelope | When |
|---------------|----------|------|
| Success | `okResponse(data, metadata)` | Pipeline completed, all agents succeeded |
| Partial | `okResponse(data, metadata)` with `warnings[]` | Some agents failed, partial results available |
| Error | `errorResponse(message, metadata)` | Bootstrap missing, graph unavailable, pipeline crashed |

All MCP tool responses include `ToolMetadata` with `last_bootstrap`, `staleness`, and `query_ms`.

## Onboarding Agent Teams Detection (ONBD-06)

Output contract for the agent teams detection flow during onboarding.

| Element | Copy |
|---------|------|
| Detection prompt | `Agent teams enable parallel execution during orient. Enable now? [Y/n]` |

## SendMessage Protocol (EXEC-08)

Structured message format for agent team communication.

### Handoff Signals

```json
{
  "type": "ready" | "done" | "blocked",
  "files": ["src/path/to/file.ts"],
  "detail": "Human-readable context"
}
```

### Discovery Signals

```json
{
  "type": "discovery",
  "category": "api_change" | "new_utility" | "pattern" | "warning",
  "detail": "Human-readable description of the discovery",
  "files": ["src/path/to/affected-file.ts"]
}
```

All signals logged to `coordination.md` regardless of execution mode.
