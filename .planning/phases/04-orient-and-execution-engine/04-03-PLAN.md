    **src/config/schema.ts**: Make `execute.parallel` optional per D-44 and Pitfall 6.

    Change the execute section from:
    ```typescript
    execute: z.object({
      parallel: z.enum(["auto", "sequential", "parallel"]),
      max_agents_concurrent: z.number().int().min(1).max(10),
    }),
    ```
    To:
    ```typescript
    execute: z.object({
      parallel: z.enum(["auto", "sequential", "parallel"]).optional(),  // D-44: deprecated, planner picks strategy. Kept optional for backward compat with existing config.yml files.
      max_agents_concurrent: z.number().int().min(1).max(10),
    }),
    ```

    This makes existing configs with the field still valid, and new configs don't need it. The field is tolerated but never read by the orient/execution engine.

    **src/config/defaults.ts**: Remove `parallel` from the execute defaults (or keep it as a comment explaining deprecation).

    Change from:

      parallel: "auto",

    execute: {

    **src/onboard/agent-teams.ts**: Agent teams detection and enablement for the onboarding flow per D-41, D-42, D-43.

    Export `detectAgentTeamsOnboard(): { status: 'already_enabled' | 'not_enabled'; source?: 'env_var' | 'settings_json' }`:
    - Check `process.env.CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` -- if '1' or 'true', return `{ status: 'already_enabled', source: 'env_var' }`
    - Check `~/.claude/settings.json` at `settings.env.CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` -- if '1', return `{ status: 'already_enabled', source: 'settings_json' }`
    - Otherwise return `{ status: 'not_enabled' }`
    - Handle missing file, parse errors gracefully

    Export `enableAgentTeams(): { success: boolean; message: string }`:
    - Read `~/.claude/settings.json` (or start with empty object `{}` if missing)
    - Parse JSON; on parse error return `{ success: false, message: 'Failed to parse ~/.claude/settings.json' }`
    - Ensure `settings.env` exists as an object
    - Set `settings.env.CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS = '1'`
    - Create `~/.claude/` directory with `fs.mkdirSync(dir, { recursive: true })` if needed
    - Write `JSON.stringify(settings, null, 2)` back to settings.json
    - Return `{ success: true, message: 'Agent teams enabled in ~/.claude/settings.json' }`
    - On write error: return `{ success: false, message: 'Failed to write ~/.claude/settings.json: {error}' }`

    Export `isAgentTeamsEnabled(): boolean`:
    - Returns true if env var is '1'/'true' OR settings.json has it set to '1'
    - Convenience wrapper used by other modules

    Export `getAgentTeamsOnboardMessage(status: 'already_enabled' | 'not_enabled' | 'enabled' | 'declined'): string`:
    - Returns the UI-SPEC copy for each state:
      - 'already_enabled': `'Agent teams already enabled. Orient will use parallel execution when the planner identifies independent tasks.'`
      - 'not_enabled': `'Agent teams enable parallel execution during orient. Enable now? [Y/n]'`
      - 'enabled': `'Agent teams enabled in \`~/.claude/settings.json\`. Orient will use parallel execution when the planner identifies independent tasks.'`
      - 'declined': `'Agent teams not enabled. Orient will run sequentially. You can enable later via \`/codescope:settings\`.'`
