    **src/verify/run-verify.ts** — CLI entry point matching run-orient.ts pattern:

    ```typescript
    #!/usr/bin/env node
    ```

    Import:
    - `runStaticVerify` from `./static-verify.js`
    - `runRuntimeVerify` from `./runtime-verify.js`
    - `writeVerifyReport` from `./report-writer.js`
    - Types from `./types.js`
    - `loadConfig` from `../config/loader.js`
    - `execSync` from `node:child_process`
    - `fs`, `path` from node

    **Argument parsing** (same pattern as run-orient.ts):
    - `--project-root` — project root directory (default: cwd)
    - `--task-slug` — task identifier (required)
    - `--task-description` — task description for report (default: taskSlug)
    - `--plan-path` — path to execution plan (required for blast radius diff)
    - `--scope-contract-path` — path to scope contract (required for scope drift + code review)
    - `--phase` — `static`, `runtime`, or omit for both sequentially
    - `--execution-dir` — execution directory for storing pre-execution HEAD SHA

    **Main function:**
    1. Parse args
    2. Get changed files: `execSync('git diff --name-only HEAD', { encoding: 'utf-8', cwd: projectRoot }).trim().split('\n').filter(Boolean)`
    3. If no changed files, output JSON `{ status: 'skipped', reason: 'No changed files detected' }` and exit 0
    4. Build stub callbacks (JSON output to stdout, no actual LLM dispatch — skill body handles that):
       ```typescript
       const callbacks: VerifyCallbacks = {
         dispatchReviewAgent: async (prompt) => {
           // Output prompt for skill body to dispatch
           console.error(JSON.stringify({ type: 'dispatch_review', prompt }));
           return '[]'; // Empty findings — skill body will re-invoke with actual results
         },
         dispatchSmokeAgent: async (prompt) => {
           console.error(JSON.stringify({ type: 'dispatch_smoke', prompt }));
           return ''; // Empty — skill body handles
         },
         onProgress: (msg) => console.error(msg),
       };
       ```
    5. Based on `--phase`:
       - `static`: Run `runStaticVerify(options, callbacks)`, output JSON result to stdout
       - `runtime`: Load config, run `runRuntimeVerify(options, callbacks)`, output JSON to stdout
       - No phase: Run static first, then runtime. Assemble VerifyReport, call `writeVerifyReport()`, output JSON with reportPath
    6. Error handling: catch, output JSON error to stderr, exit 1
