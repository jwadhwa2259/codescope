    **src/verify/server-lifecycle.ts** — Server start, readiness, and cleanup:

    Import `spawn`, `execSync`, `ChildProcess` from `node:child_process`.

    ```typescript
    export interface ServerHandle {
      process: ChildProcess;
      port: number;
      pid: number;
    }
    ```

    `export async function startServer(command: string, options: { healthCheck?: string; readySignal?: string; timeoutSeconds: number }): Promise<ServerHandle>`
    - Spawn with `{ detached: true, stdio: ['ignore', 'pipe', 'pipe'] }` to create process group
    - Extract port from healthCheck URL (parse URL, get port) or default to 3000
    - Readiness strategies (per D-15):
      1. If `healthCheck` provided: poll with `fetch(healthCheck)` every 1000ms in a while loop. Break when `res.ok`. Throw if `Date.now() > deadline`.
      2. Else if `readySignal` provided: listen on `child.stdout` 'data' event for string containing readySignal. Use Promise with setTimeout for deadline.
      3. Else: `await new Promise(resolve => setTimeout(resolve, 5000))` (5s fixed delay)
    - Return `{ process: child, port, pid: child.pid! }`

    `export async function stopServer(handle: ServerHandle): Promise<void>`
    - Step 1: Kill process group: `try { process.kill(-handle.pid, 'SIGTERM'); } catch { /* already dead */ }`
    - Step 2: Wait up to 3s for port to free. Check every 500ms with `isPortInUse(handle.port)`.
    - Step 3: If still in use after 3s, force kill: `const pid = execSync('lsof -ti :' + handle.port, { encoding: 'utf-8' }).trim(); if (pid) process.kill(parseInt(pid, 10), 'SIGKILL');` (per D-16)

    `function isPortInUse(port: number): boolean`
    - `try { execSync('lsof -ti :' + port); return true; } catch { return false; }`

    `function sleep(ms: number): Promise<void>`
    - `return new Promise(resolve => setTimeout(resolve, ms));`

    `function extractPort(urlOrCommand: string): number`
    - Try `new URL(urlOrCommand)` and return parsed port
    - Fallback: regex match for `:(\d{4,5})` in the string
    - Default: 3000

    **IMPORTANT:** Per locked decision D-14, endpoint detection MUST use web-tree-sitter AST analysis, NOT regex. This reuses the existing web-tree-sitter infrastructure already established in the bootstrap phase.

    ```typescript
    export interface DetectedEndpoint {
      file: string;
      method: string;
      route: string;
      code: string;  // the endpoint handler code
    }
    ```

    `export async function detectNewEndpoints(projectRoot: string, changedFiles: string[]): Promise<DetectedEndpoint[]>`
    - Get list of newly created files from git diff: `execSync('git diff --name-only --diff-filter=A HEAD', { encoding: 'utf-8', cwd: projectRoot })` -- filter for Added files only
    - Intersect with changedFiles (only analyze files that are both new AND in the changed set)
    - For each new file:
      1. Determine language from file extension (.ts/.tsx/.js/.jsx -> typescript/javascript, .py -> python)
      2. Initialize web-tree-sitter Parser, load the appropriate language WASM grammar
      3. Parse the file content into an AST tree
      4. Walk the AST to detect endpoint declarations by node type:

      **Express/Koa (JS/TS):** Query for `call_expression` nodes where:
        - The callee is a `member_expression` with property name matching `get|post|put|delete|patch`
        - The object name matches common patterns (app, router, server)
        - The first argument is a `string` node (the route path)
        - Extract: method from property name, route from string argument, code from the call expression text

      **Next.js App Router (JS/TS):** Query for `export_statement` nodes containing a `function_declaration` where:
        - The function name is one of: GET, POST, PUT, DELETE, PATCH, HEAD
        - The file path matches `**/route.ts` or `**/route.js`
        - Extract: method from function name, route inferred from file path (e.g., `app/api/users/route.ts` -> `/api/users`), code from function body

      **Next.js Pages Router (JS/TS):** Query for `export_statement` with `default` keyword in files under `pages/api/`:
        - Extract: method as "ALL" (pages API routes handle all methods), route inferred from file path, code from the exported function

      **Flask/FastAPI (Python):** Query for `decorated_definition` nodes where:
        - The decorator is a `call` node with attribute name matching `route|get|post|put|delete|patch`
        - The decorator object matches `app`, `blueprint`, `router`
        - The first argument is a `string` node (the route path)
        - Extract: method from decorator attribute (or "ALL" for @app.route), route from string argument, code from the function definition text

      5. Call `tree.delete()` after parsing each file to prevent memory leaks (per CLAUDE.md web-tree-sitter memory management constraint)
    - Return array of DetectedEndpoint with file, method, route, and surrounding code (the full AST node text for the endpoint handler, capped at 30 lines)

    `export function buildSmokePrompt(endpoints: DetectedEndpoint[], projectTestFramework: string): string`

      - "Use the {projectTestFramework} test framework."
      - "For HTTP endpoints: test that GET returns 200 or expected status. For auth-required endpoints, expect 401."
      - "Write the test to a temp file. The file will be cleaned up after running."
      - "Output the test code as a single code block."
