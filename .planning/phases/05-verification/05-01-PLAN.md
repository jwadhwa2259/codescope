    **src/verify/blast-radius-diff.ts** — Compute blast radius diff:

    - Import `readPlanFromDisk` from `../execution/orchestrator.js`
    - Import `getGraph` from `../graph/cache.js`
    - Import `execSync` from `node:child_process`
    - Import types from `./types.js`
    - Function signature: `export function computeBlastRadiusDiff(projectRoot: string, planPath: string, scopeContractPath: string, changedFiles: string[]): BlastRadiusDiffResult`
    - Accept `changedFiles` as parameter instead of computing git diff internally (easier to test, caller provides git diff result)
    - Step 1: Read plan from disk via `readPlanFromDisk(planPath)`, extract predicted files from all agents' `exclusiveWriteFiles` arrays into a Set
    - Step 2: Compute surprises (files in changedFiles but NOT in predictedFiles):
      - Load graph via `getGraph(projectRoot)`
      - For each surprise file, find minimum graph distance from any predicted file
      - Use graph.forEachNode to find file nodes, then BFS outward checking if surprise file appears
      - If file not in graph or unreachable, set minHopDistance = -1
      - Severity: hops 1-2 = "WARN", hops 3+ or -1 (unconnected) = "ERROR" (per D-08)
    - Step 3: Compute skips (files in predictedFiles but NOT in changedFiles set):
      - Each skip gets severity "INFO" and reason "Predicted but not modified -- may have been handled by a different approach or deemed unnecessary by execution agent" (per D-09, UI-SPEC)
    - Step 4: Scope drift (per D-10):
      - Read scope contract from disk, parse JSON
      - Extract affectedFiles array, build set of filePath values
      - Files in changedFiles but NOT in affectedFiles set = scope drift candidates (WARN)
    - Return BlastRadiusDiffResult with timing_ms
