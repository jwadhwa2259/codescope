Returns a lightweight brief with relevant files (ranked by centrality), applicable conventions, danger zones in the change area, community context, and golden files. This is NOT the full orient pipeline (that is the /codescope:orient skill). Related tools: codescope_blast_radius, codescope_conventions, codescope_recall."
    - Input schema: `{ task: z.string().describe("Task description to orient on (e.g., 'add user authentication', 'refactor error handling')") }`
    - Handler per D-27, D-28:
      1. Check isBootstrapped, return NOT_BOOTSTRAPPED if false
      2. Extract keywords from task: split by spaces, filter out stop words (the, a, an, to, for, in, of, and, or, is, are, be, with, from, by), lowercase
      3. Call `getGraph(projectRoot)` for cached graph + centralities
      4. Find matching nodes: `graph.filterNodes((n, attr) => keywords.some(kw => attr.name?.toLowerCase().includes(kw) || attr.filePath?.toLowerCase().includes(kw)))`
      5. Expand 1-2 hops: for each matching node, collect `graph.neighbors()` (1 hop) and their neighbors (2 hop)
      6. Deduplicate, rank all collected nodes by centrality (from cached centralities)
      7. Limit to top 20 results per RESEARCH.md recommendation
      8. For each result, include: `{ filePath, name, kind, centrality, community, risk_level }` where risk_level based on centrality tiers (D-23)
      9. Read conventions.md and filter to conventions relevant to the matched files
      10. Compute danger zones from matched node neighborhoods using `computeDangerZones()` from analytics
      11. Return okResponse with `{ task, relevant_files: [...], conventions: [...], danger_zones: [...], communities: [...] }`
