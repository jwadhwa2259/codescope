    export interface BuildGraphOptions {

    Internal `walkSourceFiles(rootDir: string, ignorePatterns?: string[]): string[]` function:
    - Synchronous recursive directory walk
    - Skip directories in DEFAULT_IGNORE set: `node_modules`, `dist`, `build`, `.git`, `vendor`, `.next`, `.nuxt`, `__pycache__`, `.venv`, `venv`, `coverage`, `.cache`, `.output`
    - Skip dotfiles/dotdirs (entries starting with `.` except `.` itself)
    - Skip any additional patterns from ignorePatterns parameter
    - Only include files where `detectLanguage(filePath)` returns non-null
    - Return array of absolute file paths

    `buildGraph` implementation:
    1. Record start time with `performance.now()` (or `Date.now()`)
    2. Create ParserPool instance
    3. Walk files via walkSourceFiles
    4. Create BatchWriter with `new BatchWriter(options.batchDir, "graph-builder")`
    5. For each file:
       a. Call `parseFile(filePath, pool)` to get ParseResult
       b. If null, skip (unsupported language)
       c. Compute relative path: `path.relative(options.projectRoot, filePath)`
       d. Count lines: `source.split('\n').length`
       e. Add file node via `writer.addNode({ name: path.basename(filePath), kind: "file", file_path: relativePath, language: detectLanguage(filePath), loc: lineCount, start_line: 1, end_line: lineCount, is_exported: false, is_test: filePath.includes('.test.') || filePath.includes('.spec.') || filePath.includes('__tests__') })`
       f. For each function in ParseResult.functions: add node with kind="function", file_path=relativePath, start_line, end_line, is_exported, signature=`${name}(${params.join(', ')})`; add CONTAINS edge from file to function (source_name=basename, source_file_path=relativePath, target_name=funcName, target_file_path=relativePath)
       g. For each class: add node with kind="class", same pattern; add CONTAINS edge
       h. For each variable (if exported): add node with kind="variable"; add CONTAINS edge
       i. For each import in ParseResult.imports: resolve the import path using the appropriate resolver (createTypeScriptResolver for TS/JS, resolvePythonImport for Python). If resolved to a file in the project, compute its relative path and add an IMPORTS edge (source_name=file basename, source_file_path=this file's relative path, target_name=resolved file basename, target_file_path=resolved file's relative path). If resolution fails, log in errors array but continue.
       j. For each class with extends (check node text for `extends` keyword via ClassInfo): add EXTENDS edge if the parent class can be found
    6. Call `writer.flush()`
    7. Open database, create schema, call `processBatchFiles(db, options.batchDir)`, close database
    8. Return BuildGraphResult with counts from BatchResult + filesProcessed + durationMs

    **Important implementation details per D-17:**
    - Read the source file content directly with `fs.readFileSync` for line counting (parseFile already reads it but doesn't expose source)
    - Create the TS resolver once with `createTypeScriptResolver({ projectRoot: options.projectRoot })` and reuse across all files
    - For Python files, call `resolvePythonImport(importSource, fromFile, options.projectRoot)` and only create edge if `result.resolved !== null && !result.isStdlib`
    - Per D-27: process files in reasonable order (no specific batching required since BatchWriter handles it)
    - Per Pitfall 4: use relative file_path as canonical identifier; track processed files in a Set to avoid duplicates

    **Function: `loadGraphFromSQLite(db: DatabaseType): DirectedGraph`**
    - Query all rows from `nodes` table: `SELECT id, name, kind, file_path FROM nodes`
    - Add each as node with `graph.addNode(String(node.id), { name: node.name, kind: node.kind, filePath: node.file_path })`
    - Query all rows from `edges` table: `SELECT source_id, target_id, kind, weight FROM edges`
    - For each edge, use `graph.mergeEdge(String(edge.source_id), String(edge.target_id), { kind: edge.kind, weight: edge.weight })` to handle duplicates gracefully (per Pitfall 1 from RESEARCH.md)
    - Return the graph

    **Function: `computeCentrality(graph: DirectedGraph): CentralityResult`**
    - Call `inDegreeCentrality(graph)` which returns `Record<string, number>` mapping nodeId to normalized centrality
    - Convert to Map and return as CentralityResult
    - Also assign to node attributes: `inDegreeCentrality.assign(graph, { nodeCentralityAttribute: "inDegree" })`

    **Function: `runCommunityDetection(graph: DirectedGraph, db: DatabaseType): CommunityResult`**
    - Call `louvain.detailed(graph, { resolution: 1.0 })` to get `{ communities, count, modularity }`
    - Clear existing communities: `db.prepare("DELETE FROM communities").run()`
    - Write to SQLite communities table in a transaction:
      ```
      const insert = db.prepare("INSERT INTO communities (node_id, community_id, modularity_class) VALUES (?, ?, ?)");
      const writeAll = db.transaction(() => {
        for (const [nodeId, communityId] of Object.entries(details.communities)) {
          const label = deriveCommunityLabel(graph, communityId, details.communities);
          insert.run(Number(nodeId), communityId, label);
        }
      });
      writeAll();
      ```
    - Per D-20: modularity_class is human-readable label derived from most common directory/namespace

    **Internal function: `deriveCommunityLabel(graph: DirectedGraph, communityId: number, communities: Record<string, number>): string`**
    - Collect filePath attributes from all nodes in the given community
    - Extract the first 2 path segments of each (e.g., `src/parser` from `src/parser/extract.ts`)
    - Return the most common directory path as the label
    - If no file paths found, return `"unknown"`

    **Function: `blastRadius(graph: DirectedGraph, nodeId: string, maxHops: number = 4): BlastRadiusNode[]`**
    - Use `bfsFromNode(graph, nodeId, (node, attr, depth) => { ... })` per RESEARCH.md Pattern 2
    - If `depth > maxHops`, return `true` to stop traversal
    - Classify: hop 0 = Red, hop 1 = Orange, hop 2 = Yellow, hop 3+ = Green (per GRPH-04 spec)
    - Push each visited node to results array with nodeId, name, filePath, hop, risk
    - Return results sorted by hop ascending

    **Function: `computeDangerZones(graph: DirectedGraph, centralities: Map<string, number>, communities: Record<string, number>): DangerZoneEntry[]`**
    - Per D-16: multi-signal scoring combining in-degree centrality, cross-boundary edges, file size
    - For each file node (kind === "file"):
      a. Get its in-degree centrality from centralities map
      b. Count how many distinct communities its neighbors belong to (cross-boundary signal)
      c. Get file LOC from node attributes if available
      d. Calculate riskScore = (centrality * 0.5) + (communitiesTouched / maxCommunities * 0.3) + (normalizedLoc * 0.2)
      e. Build reasons array: "High in-degree centrality ({score})" if centrality > 0.1, "Touches {N} communities" if > 1, "Large file ({LOC} lines)" if LOC > 500
    - Sort by riskScore descending
    - Return top entries (all entries -- caller decides how many to use)
