**Event emission pattern (D-33):**

Events are emitted by appending JSON lines to `.claude/codescope/events.log`. The dashboard server tails this file and broadcasts via WebSocket. This is a decoupled architecture -- pipelines do NOT import dashboard code.

Create a helper function INLINE in each file (do NOT create a shared import since these files are in different build isolation zones -- consistent with Phase 10 and Phase 12 module isolation pattern):

```typescript
function emitEvent(projectRoot: string, event: Record<string, unknown>): void {
  try {
    const eventsPath = path.join(getCodescopePath(projectRoot), 'events.log');
    fs.appendFileSync(eventsPath, JSON.stringify({ ...event, ts: new Date().toISOString() }) + '\n');
  } catch {
    // Dashboard events are observability, not critical path. Swallow errors.
  }
}
```
