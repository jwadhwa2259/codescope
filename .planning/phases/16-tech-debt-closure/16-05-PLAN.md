    Then create `.github/workflows/build-platform-packages.yml` with this exact content:

    ```yaml
    name: Build Platform Packages

    on:
      workflow_dispatch:
      push:
        paths:
          - 'package.json'
          - 'scripts/build-platform-packages.sh'

    jobs:
      build:
        strategy:
          matrix:
            include:
              - os: macos-14
                platform: darwin-arm64
              - os: macos-13
                platform: darwin-x64
              - os: ubuntu-latest
                platform: linux-x64
              - os: windows-latest
                platform: win32-x64

        runs-on: ${{ matrix.os }}
        name: Build ${{ matrix.platform }}

        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '22'

          - name: Install dependencies
            run: npm ci --ignore-scripts

          - name: Install better-sqlite3 with native build
            run: npm rebuild better-sqlite3

          - name: Run platform package build script
            run: |
              chmod +x scripts/build-platform-packages.sh
              ./scripts/build-platform-packages.sh
            shell: bash

          - name: Verify binary exists
            run: |
              ls -la platform-packages/${{ matrix.platform }}/
              file platform-packages/${{ matrix.platform }}/better_sqlite3.node || true
            shell: bash

          - name: Upload platform package artifact
            uses: actions/upload-artifact@v4
            with:
              name: platform-${{ matrix.platform }}
              path: platform-packages/${{ matrix.platform }}/better_sqlite3.node
              retention-days: 90

      collect:
        needs: build
        runs-on: ubuntu-latest
        name: Collect all platform binaries

        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Download all artifacts
            uses: actions/download-artifact@v4
            with:
              path: downloaded-artifacts

          - name: Copy binaries to platform-packages
            run: |
              for platform in darwin-arm64 darwin-x64 linux-x64 win32-x64; do
                if [ -f "downloaded-artifacts/platform-${platform}/better_sqlite3.node" ]; then
                  cp "downloaded-artifacts/platform-${platform}/better_sqlite3.node" "platform-packages/${platform}/better_sqlite3.node"
                  echo "Copied ${platform} binary"
                else
                  echo "WARNING: ${platform} binary not found"
                fi
              done

          - name: Verify all binaries
            run: |
              echo "Platform package contents:"
              for platform in darwin-arm64 darwin-x64 linux-x64 win32-x64; do
                echo "--- ${platform} ---"
                ls -la "platform-packages/${platform}/" || echo "MISSING"
              done

          - name: Upload complete platform-packages
            uses: actions/upload-artifact@v4
            with:
              name: all-platform-packages
              path: platform-packages/
              retention-days: 90
    ```

    Key design decisions:
    - `macos-14` is ARM64 (M-series), `macos-13` is Intel x64
    - `workflow_dispatch` allows manual triggering
    - `npm ci --ignore-scripts` avoids running postinstall (which would trigger build before better-sqlite3 is ready)
    - `npm rebuild better-sqlite3` forces native compilation on the target platform
    - The `collect` job downloads all 4 binaries and assembles the complete platform-packages directory
    - Artifacts are retained for 90 days for download

    # Platform Packages

    Pre-built better-sqlite3 native binaries for each supported platform. These are published as optional dependencies of the main `codescope` package.

    ## Current Status

    | Platform | Package Name | Binary | Status |
    |----------|-------------|--------|--------|
    | macOS ARM (M-series) | @codescope/better-sqlite3-darwin-arm64 | better_sqlite3.node | Built locally |
    | macOS Intel | @codescope/better-sqlite3-darwin-x64 | better_sqlite3.node | Requires CI build |
    | Linux x64 | @codescope/better-sqlite3-linux-x64 | better_sqlite3.node | Requires CI build |
    | Windows x64 | @codescope/better-sqlite3-win32-x64 | better_sqlite3.node | Requires CI build |

    ## Building Binaries

    ### Local (current platform only)

    ```bash
    ./scripts/build-platform-packages.sh
    ```

    This detects the current platform and architecture, finds the better-sqlite3 native binary from node_modules, and copies it to the appropriate platform-packages subdirectory.

    ### CI (all platforms)

    The GitHub Actions workflow at `.github/workflows/build-platform-packages.yml` builds binaries on all 4 platforms using a matrix strategy:

    1. **Trigger manually:** Go to Actions > Build Platform Packages > Run workflow
    2. **Auto-trigger:** Pushes that modify `package.json` or `scripts/build-platform-packages.sh`
    3. **Download:** After the workflow completes, download the `all-platform-packages` artifact

    ### Runner Matrix

    | Runner | Platform |
    |--------|----------|
    | macos-14 | darwin-arm64 (M-series) |
    | macos-13 | darwin-x64 (Intel) |
    | ubuntu-latest | linux-x64 |
    | windows-latest | win32-x64 |

    ## Package Structure

    Each platform directory contains:
    - `package.json` — npm package metadata with `os` and `cpu` fields
    - `better_sqlite3.node` — Native binary (after build)

    ## Publishing

    After building all binaries:

    ```bash
    for dir in darwin-arm64 darwin-x64 linux-x64 win32-x64; do
      cd platform-packages/$dir
      npm publish --access public
      cd ../..
    done
    ```

    ## Fallback

    If a platform package is unavailable, the main `codescope` package falls back to compiling better-sqlite3 from source (requires build tools). The native loader in `src/native-loader.ts` warns but does not exit on missing platform packages.
    ```
