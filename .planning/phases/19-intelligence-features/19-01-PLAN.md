  <behavior>
    Reference Index (reference-index.ts):
    - buildReferenceIndex(db, codescopeDir) returns ReferenceIndex with generated timestamp and files map
    - Each file entry has referencePath (string), roleLabel (string), score (number 0-1)
    - Noise files (isNoiseFile() returns true) are excluded from both candidates and results
    - Files are scoped by role: utility files match against utility files, route-handler against route-handler (per D-03)
    - For "general" role files (low confidence), compare against all non-noise files (per Open Question 1)
    - Similarity uses 4 weighted signals per D-02: convention density 40%, community proximity 25%, directory proximity 20%, shared imports 15%
    - Convention density: ratio of HIGH-CONF conventions a file has vs max in its role group (read from conventions.md)
    - Community proximity: 1.0 if same community ID, 0.0 otherwise (from runCommunityDetection)
    - Directory proximity: 1.0 minus (normalized path distance / max depth), computed from path.dirname
    - Shared imports: count of shared import targets between two files / max(imports of either) (from edges table)
    - Files with no same-role peers get no entry (empty is valid)
    - A file cannot reference itself

    Violation Index (violation-index.ts):
    - buildViolationIndex(db, codescopeDir) returns ViolationIndex with generated timestamp and files map
    - Each file entry is an array of ViolationEntry: { ruleId: string, detected: string, expected: string, line: number }
    - Only HIGH-CONF conventions (confidence === "HIGH-CONF") trigger entries per D-11
    - Type name checking (VALID-02): extract type/interface names from graph nodes (kind='type' or kind='interface'), store known types. Files referencing unknown type names get a violation entry.
    - Import path checking (VALID-03): extract resolved import targets from edges table (kind='imports'). Files with import edges to non-existent targets get a violation entry.
    - Files with zero violations get no entry in the map (sparse)
  </behavior>

    /** Per-file entry in the reference index. */
    export interface ReferenceFileEntry {
      /** Path to the most similar same-role file. */
      referencePath: string;
      /** Human-readable role label (e.g., "utility", "route-handler"). */
      roleLabel: string;
      /** Similarity score (0-1). */
      score: number;
    }

    /** Top-level reference index structure. */
    export interface ReferenceIndex {
      /** ISO 8601 timestamp of when this index was generated. */
      generated: string;
      /** Per-file reference data, keyed by relative file path. */
      files: Record<string, ReferenceFileEntry>;
    }

    // ---- Violation Index ----

    /** Single violation entry for a file. */
    export interface ViolationEntry {
      /** Rule or check that detected the deviation. */
      ruleId: string;
      /** What was actually found in the file. */
      detected: string;
      /** What was expected based on conventions. */
      expected: string;
      /** Line number where the deviation was found (0 if file-level). */
      line: number;
    }

    /** Top-level violation index structure. */
    export interface ViolationIndex {
      /** ISO 8601 timestamp of when this index was generated. */
      generated: string;
      /** Per-file violation lists, keyed by relative file path. Sparse: files with no violations are omitted. */
