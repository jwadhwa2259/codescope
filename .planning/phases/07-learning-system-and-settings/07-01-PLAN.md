    **src/learning/parser.ts:** Parse and serialize learnings.md entries. Use regex to split `## Entries` section by `### ` headings (matching existing format in learning-synthesizer.ts lines 70-79 and ignore-filter.ts lines 113-125). Extract metadata fields using `**Field:** value` regex pattern. Frontmatter parsing: extract YAML between `---` delimiters using regex, parse key-value pairs. Preserve non-Entries sections (like `## Ignore Patterns`, `## Schema`) as raw strings in a Map for lossless roundtrip. `serializeLearnings` reconstructs the full file: frontmatter + `# Learnings` + `## Schema` + `## Entries` + each entry via `serializeLearningEntry` + preserved sections. `serializeLearningEntry` outputs the `### title` + `- **Status:**` + `- **Type:**` + `- **Discovered:**` + `- **Expires:**` + `- **Evidence:**` format, plus optional fields (note, pattern, scope, criterion, file, severity, contradicts) when present. Update `total_learnings` in frontmatter to match actual entry count on serialize.

    **src/learning/decay.ts:** Pure functions. `computeExpiry(type, discoveredDate, decayConfig)`: use `Date.setDate(Date.getDate() + days)` for correct month/year rollover per Research Pitfall 3. Gotchas use `decayConfig.gotchas` days, decisions and patterns use `decayConfig.decisions` days. IGNORE and TODO types return empty string (no expiry). Return ISO date string YYYY-MM-DD. `isExpired(entry, now)`: compare `new Date(entry.expires)` <= now at day granularity (truncate both to midnight). Return false if expires is empty. `runDecay(entries, decayConfig, now)`: map over entries, mark entries with status UNVERIFIED or CONTRADICTED as EXPIRED if past their expiry date. Do NOT modify IGNORE, TODO, or VERIFIED entries (IGNORE/TODO have no decay per D-09/D-10; VERIFIED entries are confirmed by user and persist indefinitely).

    **src/learning/cap.ts:** `countActiveEntries(entries)`: return entries.length (ALL types count per D-13). `enforceCapWithEviction(currentEntries, newEntries, maxActive)`: iterate new entries. If under cap, push to current. If at cap, find the EXPIRED entry with the earliest `discovered` date and evict it. If no expired entries to evict, add to `skipped` array (per D-12: "skip adding the new learning and note it in the pipeline summary"). Return `CapResult` with final entries, evicted list, skipped list.

    **src/learning/contradiction.ts:** Per D-06/D-07/D-08. `buildContradictionPrompt(newLearning, existingLearning)`: returns a strict LLM prompt: "Do these two learnings DIRECTLY contradict each other? A contradiction means adopting both is impossible. Being related or about the same topic is NOT a contradiction. Return JSON: {isContradiction: boolean, reason: string}. Learning A: {newLearning.title} - {newLearning.evidence}. Learning B: {existingLearning.title} - {existingLearning.evidence}." `checkContradictions(newEntry, existingEntries, llmCallback?)`: first, do a simple heuristic check — look for antonym patterns in titles (e.g., "use X" vs "avoid X", "prefer X" vs "do not use X"). If heuristic detects a likely contradiction, flag it without LLM. For non-obvious cases and when llmCallback is provided, call `llmCallback(buildContradictionPrompt(...))` for each existing entry and parse the JSON response. Return array of `ContradictionResult`. The llmCallback parameter is optional — when not provided (e.g., during CLI batch processing without LLM access), only heuristic checking runs. Per D-08: code-first validation via ast-grep is done at the synthesizer agent level (it has project context), not in this pure function module.

    **src/learning/manager.ts:** High-level orchestrator. `loadLearnings(projectRoot)`: read learnings.md from `path.join(getCodescopePath(projectRoot), "learnings.md")`, call `parseLearnings(content)`, return ParsedLearnings. If file does not exist, return empty ParsedLearnings with default frontmatter. `saveLearnings(projectRoot, parsed)`: call `serializeLearnings(parsed)`, write to learnings.md. Update frontmatter.total_learnings to match entries.length. `addLearnings(projectRoot, newEntries, decayConfig, maxActive, llmCallback?)`: (1) loadLearnings, (2) runDecay on existing entries, (3) for each new entry call checkContradictions — if contradicted, set entry.status = "CONTRADICTED" and entry.contradicts = result.contradicts, (4) call enforceCapWithEviction(decayedEntries, newEntries, maxActive), (5) saveLearnings, (6) return { added: entries that made it in, skipped: entries that could not be added, contradicted: entries marked CONTRADICTED, evicted: entries removed }. Import getCodescopePath from `../utils/paths.js`.

    **src/learning/global-enrichment.ts:** Per D-21 3-strike rule. `detectRepeatedIgnores(entries, threshold = 3)`: filter entries to type === "ignore", group by `criterion + pattern` composite key, count unique `evidence` values (representing different pipeline runs — each IGNORE entry has a different taskSlug in its Context field). Return groups where count >= threshold. `buildEnrichmentUpdates(repeatedPatterns, existingGlobalIgnores)`: for each repeated pattern not already in existingGlobalIgnores, create a `GlobalEnrichmentEntry` with type "ignore_pattern", value = `{criterion}:{pattern}`, source = "auto-detected from {count} pipeline runs", recordedDate = today's ISO date. Return the array. Deduplication: compare by `criterion + pattern` composite key against existing global ignore patterns.
