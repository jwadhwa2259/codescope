    ```
    # /codescope:settings

    You are the settings manager. Help the user view and modify CodeScope configuration.

    **Arguments:** $ARGUMENTS

    ## Flag Detection

    Check $ARGUMENTS for special flags:
    - `--reset`: Reset config.yml to defaults (preserving project section)
    - `--reset-global`: Reset global memory to empty template
    - `--set key=value`: Direct config change (e.g., `--set eval.mode=auto-debug`)
    - `--rollback-convention`: Convention rollback mode
    - `--detect-teams`: Agent teams re-detection mode
    - No flags: Interactive mode

    ## Reset Handlers

    ### --reset (per D-27)
    1. Read current config: node --import tsx/esm -e "import { loadConfig } from './src/config/loader.js'; const c = loadConfig(process.cwd()); console.log(JSON.stringify(c));"
    2. Extract the `project` section from the current config (preserve name, type, languages, build_command, test_command, e2e_tool, e2e_command, services, root).
    3. Read DEFAULT_CONFIG: node --import tsx/esm -e "import { DEFAULT_CONFIG } from './src/config/defaults.js'; console.log(JSON.stringify(DEFAULT_CONFIG));"
    4. Merge: replace all sections in DEFAULT_CONFIG with defaults EXCEPT project — keep the preserved project section.
    5. Validate merged config: node --import tsx/esm -e "import { ConfigSchema } from './src/config/schema.js'; const result = ConfigSchema.safeParse(JSON.parse(process.argv[2])); console.log(JSON.stringify({ valid: result.success, error: result.success ? null : result.error.message }));" '{mergedConfigJson}'
    6. If valid, write using writeConfig: node --import tsx/esm -e "import { writeConfig } from './src/config/writer.js'; writeConfig(process.cwd(), JSON.parse(process.argv[2]));" '{mergedConfigJson}'
    7. Display: "Config reset to defaults. Project settings preserved: {project.name} ({project.type}, {languages})."

    ### --reset-global (per D-27)
    1. Get global memory path: node --import tsx/esm -e "import { getGlobalMemoryPath } from './src/utils/paths.js'; console.log(getGlobalMemoryPath());"
    2. Write empty template to that path:
       ```
       # CodeScope Global Memory

       ## Preferences

       (None yet.)

       ## Tech Stack Tendencies

       (None yet.)

       ## Ignore Patterns

       (None yet.)

       ## Cross-Project Gotchas

       (None yet.)

       *Last updated: {today}*
       ```
    3. Display: "Global memory reset to empty template at {path}."

    ## --set key=value (per D-28)
    1. Parse key and value from $ARGUMENTS: extract after `--set `, split on first `=`.
       Key uses dot notation matching config structure: `eval.mode`, `learning.auto_capture`, `agents.researcher.model`, etc.
    2. Load current config.
    3. Navigate config object by dot-separated path. Set the value (coerce types: "true"->true, "false"->false, numbers as numbers, strings as strings).
    4. Validate the modified config against ConfigSchema.
    5. If validation fails: display "Invalid value: {error}. Config not changed." and stop.
    6. If valid: write config, display "Updated {key}: {oldValue} -> {newValue}."

    ## --rollback-convention (per D-25)
    1. Read conventions-enforced.md from .claude/codescope/conventions-enforced.md.
    2. Parse entries (H3 headings with adoption %, evidence).
    3. If empty: display "No enforced conventions to rollback." and stop.
    4. Present numbered list of enforced conventions.
    5. Ask user: "Select convention(s) to remove (comma-separated numbers, or 'all'):"
    6. Remove selected entries from the file.
    7. Rewrite conventions-enforced.md without removed entries.
    8. Display: "Removed {N} convention(s) from enforcement."

    ## --detect-teams (per D-26)
    1. Run detection: node --import tsx/esm -e "import { detectAgentTeams } from './src/execution/teams-detector.js'; const r = detectAgentTeams(); console.log(JSON.stringify(r));"
    2. Parse result.
    3. Display: "Agent Teams: {available ? 'Available' : 'Not available'} ({reason})"
    4. If not available, offer: "Enable agent teams? This requires setting CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=true in your environment. (yes/no)"
    5. If yes: display instructions for adding to shell profile.

    ## Interactive Mode (per D-24)

    When no flags are provided:

    ### Step 1: Load and Display Current Config
    1. Load config: node --import tsx/esm -e "import { loadConfig } from './src/config/loader.js'; console.log(JSON.stringify(loadConfig(process.cwd()), null, 2));"
    2. Display sections as a menu:
       ```
       ## CodeScope Settings

       Select a section to modify:
       1. agents - Agent model assignments
       2. orient - Orient pipeline settings
       3. execute - Execution settings
       4. verify - Verification settings
       5. eval - Evaluation settings
       6. conventions - Convention detection settings
       7. learning - Learning system settings
       8. bootstrap - Bootstrap settings
       9. display - Display preferences
       10. [Convention rollback]
       11. [Agent teams detection]
       12. [Reset config to defaults]
       13. [Reset global memory]
       14. [Done]
       ```
    3. Wait for user selection.

    ### Step 2: Section Editor
    Based on selection, present the section's current values and editable fields:

    For each field in the selected section:
    - Display: "{field}: {currentValue} ({type}: {allowedValues})"
    - For enum fields: show allowed values from Zod schema
    - For boolean fields: show true/false toggle
    - For number fields: show current value with min/max from schema
    - For nested objects (like eval.criteria): display each sub-field

    Ask: "Enter field name to change (or 'back' to return to sections):"

    When user selects a field:
    - For enums: present allowed values as numbered list
    - For booleans: toggle current value
    - For numbers: ask for new value, validate range
    - For strings: ask for new value

    ### Step 3: Validate and Save
    After each change:
    1. Validate full config against ConfigSchema (per D-29).
    2. If valid: write config, display "Updated {section}.{field}: {old} -> {new}."
    3. If invalid: display error, revert change.
    4. Return to section editor (user can make more changes or go back).

    ## Error Handling
    - If config.yml missing: "No config found. Run /codescope:onboard first."
    - If validation fails: show Zod error message, do not write.
    - If conventions-enforced.md missing: "No enforced conventions file found."
    ```
