## Cross-Phase Integration (26/27 wired)

### Integration Findings

**1. violation-index.ts noise filter inconsistency (low severity)**
- `src/artifacts/violation-index.ts:57` defines a local `isNoiseFile()` that only filters by file extension (`.d.ts`, `.json`, `.yml`, etc.)
- `src/artifacts/reference-index.ts:20` uses the canonical `isNoiseFile` from `src/conventions/golden-files.ts` which also covers `.test.ts`, `.spec.ts`, config, generated, and deprecated files
- **Compensating control:** `classifyFileRole` post-filter at line 135 excludes test/config/deprecated roles, largely closing the gap
- **Residual risk:** A file classified as `general` role with a `.test.` fragment could slip through (low probability)
- **Affects:** VALID-04 (false positive rate), REF-03 (noise exclusion parity)
- **Fix:** Replace local function with `import { isNoiseFile } from "../conventions/golden-files.js"`

**2. Convention scanner / graph builder parallel pipelines (informational)**
- `runConventionScan` (runner.ts) operates independently of `graph.db` — uses filesystem file counts, not graph-aware counts
- The two pipelines converge at artifact generation time (`generator.ts`) when `buildReferenceIndex` reads both graph and convention data
- **Not a break** — intentional architecture. Convention adoption percentages reflect all source files, not just graph-resolved ones.
